# MrChat 调研目录

目标：在不牺牲性能的前提下，把“更好的对话体验”做成一个可运营的聚合镜像站（多模型、多租户、配额/计费、审计、可观测）。

本目录用于沉淀调研结论，要求可以直接反哺设计与实现，而不是“功能清单式复读”。

## 目录约定

- `projects/`：按项目沉淀（每个参考项目 1 份）
- `topics/`：按专题沉淀（前端性能、流式协议、数据模型等）
- `notes/`：原始材料与临时记录（例如外部报告、会议纪要）

## 调研模板（建议每份都覆盖）

- 项目/专题定位：解决什么问题，适配什么场景
- 我们关心的关键点（优先级）
- 方案概览（架构、数据流、关键模块）
- 前端：渲染策略、流式体验、长对话性能、交互设计亮点
- 后端：协议兼容、路由/重试/限流、计费/日志、缓存/DB/索引
- 可观测：指标、日志、追踪/剖析（profiling）
- 借鉴点：我们要直接拿走的 3-10 个点（带落地方式）
- 避坑点：明确不做/不建议做的点（原因）
- 待验证问题：需要跑基准/压测/读源码/做 PoC 才能定的点
